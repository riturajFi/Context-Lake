Request lifecycle

From external write to governed agent context

01

Ingest

Thin API validates auth, tenant, and idempotency. A single transaction writes the domain row, ingestion record, idempotency key, and outbox row.

02

Publish

Outbox relay polls pending rows and publishes versioned envelopes to Kafka, preserving request, trace, and tenant metadata end to end.

03

Project

Stream processor validates envelopes, deduplicates repeated delivery, and upserts tenant-scoped materialized views for low-latency reads.

04

Serve

Context query API assembles agent-friendly payloads from projections and bounded audit references instead of exposing raw internal tables.

05

Audit

Audit writer consumes audit and agent events, then persists immutable rows for compliance review, trace reconstruction, and operator debugging.
